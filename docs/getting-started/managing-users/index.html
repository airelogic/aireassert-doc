    Managing Users

    Feb 15, 2023

    Giving a user access to AireAssert ??

    • Visit the policy site for your AireAssert instance (e.g. dev) and log in with the ‘AirePolicy Root’ tenancy;
    • Find your AireAssert Tenant in the list and select ‘Users’;
    • A user will need to be invited if they are not known to airepolicy, or can be simply added if they are. The easiest way to do this is to try and add your user first, if this fails then invite them instead;
    • Your user should now be able to login to AireAssert with their google account.

    Configuring Repository Access for a User

    • Login to the AireAssert UI as an administrator;
    • Visit the ‘Manage Users’ section;
    • Select the row for the user you wish to configure repository access (click the row not the ID link!) - the row will highlight in blue;
    • In the righthand bar, select the repositories you wish your user to have access to.
