    Using tags ?? (how do i run them section)

    Jul 26, 2023

    AireAssert TestSets can very quickly get big, slow and unwieldy. As your collection of tests grows it can be really useful to run subsets of them. This is where tags come in.

    How do I tag a Scenario or Feature?

    To add a Feature level tags you can add them in the very first line of the .feature file. Adding the tags as below will run every test within the feature file:

    @MyTag
Feature: An exciting test feature
An interesting test description.

Scenario: My Test
Given something
When something else
Then something happened

Scenario: My other Test
Given something
When something else
Then something happened

    However to add a tag to a scenario simply add it to the line above that specific Scenario:

    @MyTag
Scenario: My Test
Given something
When something else
Then something happened

    This will only run the scenarios that have that specific tag on.

    Can I mix and match tags?

    Simply put yes you can! So having the same tag on .feature A and scenarios B and C on different features is completely fine. Just build up your groups of tests to match your needs.

    What tags should I use?

    You can make up any tags you like! You might wish to organise by form or business area. A couple of tags are generally useful though:

    • @SmokeTest: This should be a small collection of fast tests that are designed to quickly check the basics of the system. You can therefore run them often, and put them into pipelines easily;
    • @CriticalPath: These are the scenarios that the system needs in order to perform its primary task. If these all pass then the system should be usable, if not without issues.
    • @WIP: For your work in progress
    • @SomeBusinessArea e.g. @LungDiseaseTests: Grouping tests by a logical business area is one of the most common tag type you will see. This means you could have multiple scenarios across a variety of features with this tag.

    OK, I’ve tagged up my features, how do I run them?

    To run just a specific tag’s scenarios is simple: In the terminal Simply provide a tags parameter to the runtests call:

    ./RunTests.ps1 -tags "@MyTag"
