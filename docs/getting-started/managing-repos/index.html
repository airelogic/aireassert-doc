    Managing Repositories

    Apr 18, 2023

    This page will guide you through the process of adding or modifying Git Repositories to your AireAssert installation, which can be used to store TestSets and as a source to run them from.

    Note: You must be an administrator to add or link repositories

    Setting up a repository for linking to AireAssert

    • You will need a bitbucket repository
    • Create or find an appropriate OAuth Consumer for your repository. See Create an OAuth Consumer. Note that these permissions are set at the workspace level, and that you will need administrator access to create/view them.

    Linking your repository

    In AireAssert, visit the ‘Manage Repositories’ section. You will see a list of linked repositories and an add new form. Enter your repo details here and it should appear both in the repository list:

    Then on the left side bar:

    You can enter a default branch, but you can always switch branches later in the UI so generally this can be left blank.

    Assigning users to repositories

    Just adding a repo to the system is insufficient for your users to be able to interact with a TestSet; they must be linked to the repo. You can do this in the Manage Users screen. Simply select your desired user and you can choose which repos to assign them in the right-hand sidebar:
