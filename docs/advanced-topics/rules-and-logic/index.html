    Rules and logic

    Jul 26, 2023

    Form control names

    When testing a feature file in AireAssert there are a few things to look out for, certain symbols can not be used and will result in a test failure. The symbols which can not be used in a feature file include questions marks (?) and apostrophes (‘).

    When testing a form in AireAssert, there may be question marks in the form but should not be used in the gherkin testing .feature file as this will result in a failure for example,

    Scenario: Complete Form 
Given we are completing the testing eform 
When we set 'Testing?' to 'Yes' 
Then 'control-1' is not empty

    In order for this test to pass the Scenario should be written as seen below,

    Scenario: Complete Form 
Given we are completing the testing eform 
When we set 'Testing' to 'Yes'
Then 'control-1' is not empty

    This process is the same for apostrophes, all other characters and symbols are okay to be used in the feature file as normal. Such as,

    Scenario: Complete Form 
Given we are completing the testing eform 
When we set 'Testing!' to 'Yes' 
Then 'control-1' is not empty

    Another option is to use the building block and control name to reference the a control:

    When we set 'building-block-name-control-name' to 'Yes'
